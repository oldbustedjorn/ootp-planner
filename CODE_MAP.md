# OOTP Planner — Code Map

This is a lightweight reference to the main modules, classes, and functions.
It is intentionally curated rather than exhaustive.

---

## Top-Level Files

### `config.toml`
Purpose:
- default paths
- scoring weights
- position blends
- thresholds / tuning values

### `compare_headers.py`
Purpose:
- compare old/new CSV headers
- identify added/removed/renamed export fields between OOTP versions

## Package: `ootp_opt.ingest`

### `ootp_opt.ingest.pt_hitters`
Purpose:
- load PT hitter CSV exports
- normalize columns across OOTP versions

Key function:
- `load_pt_cards_csv(path)`

Important behavior:
- maps raw hitter export columns into stable normalized names
- handles OOTP 26 / 27 fielding duplicate-column differences

### `ootp_opt.ingest.pt_pitchers`
Purpose:
- load PT pitcher CSV exports
- normalize columns across OOTP versions

Key function:
- `load_pt_pitchers_csv(path)`

Important behavior:
- handles `HRR` / `HRA`
- handles `P.1` / `P_1`
- creates normalized pitcher scoring inputs

---

## Package: `ootp_opt.domain`

### `ootp_opt.domain.rating`
Purpose:
- core baseball scoring logic

Key classes:
- `RatingWeights`
- `HitterRoleWeights`
- `PitcherRatingWeights`
- `PitcherRoleWeights`

Key functions:
- `rate_hitters_basic(df, weights)`
- `rate_pitchers_basic(df, weights)`
- `add_hitter_and_position_scores(df, weights)`
- `add_pitcher_role_scores(df, weights)`

Important outputs produced here:
- hitter batting scores
- hitter position scores
- pinch run score
- pitcher starter/reliever role scores

---

## Package: `ootp_opt.services`

### `ootp_opt.services.rating_service`
Purpose:
- orchestration layer for scoring

Key function:
- `rate_cards_service(input_path, profile, config=None)`

Responsibilities:
- build weight objects from config
- call ingest functions
- call scoring functions
- sort scored data
- generate hitter shortlist output

### `ootp_opt.services.shortlist_service`
Purpose:
- build hitter shortlist views

Key function:
- `generate_hitter_shortlists(df, top_n=15)`

Responsibilities:
- top players per position
- top bats vs handedness
- playable positions summary
- best positions by score summary

## Naming Conventions to Preserve

### Hitters
- `batting_score_*` = offense-only scores
- `score_SS_*`, `score_CF_*`, etc. = positional scores
- `pinch_run_score` = bench-role score

### Pitchers
- `starter_score_*`
- `reliever_score_*`

Keep these consistent in future work.

---

## Important Current Rule
Do not build downstream tools (optimizer, GUI, upgrade search) in ways that duplicate scoring logic.

Everything should reuse:
- normalized ingest
- domain scoring
- roster evaluation logic (future)
